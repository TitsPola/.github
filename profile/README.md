## How We Work

<div align="center">

| Principle | What it means |
|:---------:|:--------------|
| **Ship small** | Short-lived branches, focused pull requests, frequent releases |
| **Review everything** | Every change goes through code review before it lands |
| **Automate it** | CI/CD pipelines handle testing, builds, and deployments |
| **Own it end-to-end** | The team that builds a service runs it in production |
